Applied:  2026-07-06

Files modified:

  src/styles/global.css

Changes applied (per Emil's review checklist):

  1. Press feedback on :active
     - Added .btn-interact:active { transform: scale(0.97) }
     - Every pressable element must feel responsive; scale confirms
       the UI heard the user.
     - Gate: @media (hover: hover) and (pointer: fine) wraps hover
       to prevent false positives on touch devices.

  2. Tightened transition duration
     - Before: 300ms (var(--transition-hover))
     - After:  160ms cubic-bezier(0.23, 1, 0.32, 1)
     - Emil's rule: button press feedback = 100-160ms.
       300ms on a button feels sluggish.

  3. Strong custom easing curve
     - cubic-bezier(0.23, 1, 0.32, 1) — Emil's recommended
       "strong ease-out". Built-in CSS easings are too weak
       for punchy UI feedback.

  4. Only transform and opacity animate
     - Transition scoped to transform and opacity only —
       avoids layout/paint work, runs on GPU.
     - Emil's rule: never use transition: all.

  5. Elements not modified:
     - Menu.jsx (.rice-card-btn, .menu-cta-btn) — unchanged
       (user rejected previous animated stripe approach)

--- Scroll-Active Nav Highlighting ---
Applied:  2026-07-07

Added IntersectionObserver-based section tracking so the navbar highlights
the "Events" or "Contact" link depending on which section is in the viewport
while scrolling the EventsContact page.

New files:
  src/context/ActiveSectionContext.jsx — shared context (activeSection,
    setActiveSection, clearActiveSection) consumed by Navbar + MobileNav

Modified files:
  src/App.jsx                              — wrapped app in ActiveSectionProvider
  src/pages/EventsContact.jsx              — IntersectionObserver on both
    sections, updates context on scroll
  src/components/layout/Navbar.jsx          — overrides :active state for Events
    and Contact links based on activeSection when on /events
  src/components/layout/MobileNav.jsx       — same scroll-based override in
    the mobile drawer

How it works:
  - EventsContact.jsx observes #events-room-section and #events-contact-section
    with IntersectionObserver (thresholds: 0.1/0.3/0.5/0.7/0.9)
  - The section with the highest intersection ratio (>10%) wins
  - On unmount, clearActiveSection() resets state so other pages use normal
    route-based :active
